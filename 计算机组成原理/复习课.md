### 第一章：性能计算（CPI、MIPS、CPU时间）
我们今天加快一点速度，主要是把课后习题给大家梳理一下。第一章的考试题型肯定会有选择题，这个大家自己下去复习。大题方面，基本都是大家平时作业里做过的同类题型。

大家必须要掌握的三个核心计算：
1. **CPI 怎么计算**
2. **CPU 执行时间（指令执行时间）怎么计算**
3. **MIPS 怎么计算**

以第一章的课后作业题为例：题目问优化前和优化后的 CPI 分别是多少？
CPI 的计算非常简单，就是**各类指令的 CPI 乘以它们在程序中所占的比例然后求和**。
* **优化前**：比例都已知，直接算：$1 \times 40\% + 2 \times 20\% + 2 \times 15\% + 2 \times 25\%$。
* **优化后**：题目说 A 类指令的数量减少了一半。这意味着 A 类指令的绝对数量减少了，原本占 40%，现在减半相当于只占原总数的 20%。注意，A 类指令减半后，**总的指令条数也减少了 20%**。所以优化后计算时，分母不能再是原来的总数，各项的比例需要重新除以现在的总指令比例（即 80%），也就是除以 $(20\% + 20\% + 15\% + 25\%)$。

接下来算 **MIPS**。MIPS 的计算公式是：$MIPS = \frac{f}{CPI \times 10^6}$。
很多同学做错是因为忘记乘以 $10^6$。书上的单位有时用兆赫兹（MHz）直接算，容易弄混。我要求大家统一将时钟频率 $f$ 转换为赫兹（Hz），然后在分母上务必乘以 $10^6$。如果算出来的结果是 $13312.5 \times 10^6$ 这种格式肯定是错的，单位一定要统一。

**第一题的结论分析**：
批改作业时我发现，大家前面计算基本没错，但最后得出的结论说不清楚。优化后系统的性能肯定是提升的。但是大家发现没有？我们平常总以为“CPI 越低越好，MIPS 越高越好”，但在优化后，MIPS 反而降低了，CPI 反而升高了。这说明什么？结论就是：**我们不能单纯地依靠 MIPS 或 CPI 来衡量系统的性能。** 衡量系统性能要看“三要素”：指令条数、时钟周期、CPI。光看其中一个指标是没法准确衡量性能的。

**第一章的第二道作业题（Amdahl定律/浮点指令优化）**：
如果要让程序的整体执行速度提高到两倍（即执行时间缩短一半），浮点运算指令的 CPI 应该怎么改进？
CPU 执行时间 = 指令条数 $\times$ CPI $\div$ 时钟频率（或 $\times$ 时钟周期）。
我们代入数据计算，把原执行时间缩短一半，假设新的执行时间是 0.128 秒。我们设浮点指令新的 CPI 为 $CPI_{new}$，列出等式去解。结果你们会发现，算出来的 $CPI_{new}$ 小于零！
出现负数说明什么？说明**仅靠优化浮点指令是无法实现整体性能提升两倍的**。
如果要改进，该怎么改？我们讲过一个原则叫**“加速大概率事件”**。如果要改进，肯定要优先优化占比最高、使用最频繁的指令（比如整数运算指令），或者原本 CPI 就很大的指令（比如访存指令），这样才有改进的空间。

### 第二章：数据表示（原码、反码、补码、浮点数 IEEE 754）
第二章最核心的内容就是**带符号整数的表示**和**浮点数的表示**。

**1. 原码、反码、补码的转换**
* **正数（如 +127）**：8位表示中，最高位为0，原码、反码、补码都一样，都是 `01111111`。
* **负数（如 -127）**：数值部分写出来，原码是最高符号位为1（`11111111`）；反码是符号位不变，数值位按位取反（`10000000`）；补码是反码加1（`10000001`）。
* **记住一个规律**：不管 8位、16位、32位还是 64位，**-1 的补码永远是全 1**（即 16进制的全部是 F）。
* **最小负数（如 8位的 -128）**：-128 无法用 8位原码或反码表示（数值位不够），但可以用补码表示。8位补码最小能表示 $-2^7 = -128$。记住规律：**最小的负数补码永远是 1 后面跟若干个 0**（如 `10000000`）；**最大的正数永远是 0 后面跟若干个 1**。

**2. 补码求真值（十进制数）**
给你一个负数的补码，求它对应的十进制数，有两种方法：
* **方法一（取反加一法）**：对这个补码连同符号位取反加1，得到它对应正数的补码，算出来再加个负号。我教过大家一个简便操作：从右往左找，**找到第一个 1，这个 1 和它右边的 0 保持不变，它左边的所有位（包括符号位）全部取反**，这就相当于取反加 1 了。
* **方法二（位权展开法，推荐）**：只要看到最高位符号位是 1，就直接将这一位当作负的权值计算。比如 8位补码，最高位权值就是 $-2^7$，后面的数值位照常加上正的权值（$+ 2^6 + 2^5...$）。这种方法更不容易出错。

**3. IEEE 754 浮点数表示**
必须掌握书上的例子（将十进制小数转换为 IEEE 754 单精度浮点数，以及反向转换）。
* **机器数转十进制**：拆分出 1位符号位（数符）、8位阶码、23位尾数。注意阶码要**减去 127**（移码转真值），尾数前面要加上**隐含的 "1."**。最后写成 $(-1)^S \times 1.M \times 2^{E-127}$ 进行计算。
* **十进制转机器数（如 8.25）**：
  1. 先转成二进制：$8.25 = 1000.01$。
  2. 规格化：移动小数点，变成 $1.00001 \times 2^3$。
  3. 确定符号位：正数，$S=0$。
  4. 计算阶码：$E = 3 + 127 = 130$，转成二进制 `10000010`。
  5. 处理尾数：取小数点后面的部分 `00001`，并在后面补 0 凑够 23 位。
  6. **强调**：考试时如果最后结果没有将这 32 位二进制转换为 **16 进制**形式表示，是要扣分的！不要让我去数那一长串的 0 和 1。

### 第三章：算术运算（加减法、乘除法、浮点运算）
这里主要是运算规律和溢出判断。

**1. 十六进制补码的快速减法**
比如给你一个数 `FFFFFFDF`，让你求它对应的正值。最快的方法是用全 F 减去它再加 1：`FFFFFFFF - FFFFFFDF + 1 = 00000020 + 1 = 21H`。所以它代表的真值就是 `-21H`。
求 `X - Y`，就等于 `X 的补码 + (-Y)的补码`。求 `-Y` 的补码，同样可以用全 F 减去 Y 再加 1 来快速获得。

**2. 移位与溢出**
* **无符号数**：左移右移都是添 0。
* **有符号数（补码）**：左移末位添 0（可能会改变符号导致溢出）；右移高位**添符号位**（正数添 0，负数添 1），这叫算术右移。
* **溢出判断**：什么时候会溢出？只有**同号相加**（正加正变负，负加负变正）才会溢出。异号相加（正数加负数）绝不可能溢出。

**3. 变形补码（双符号位法）**
变形补码用两位表示符号：`00` 表示正，`11` 表示负。
如果加法运算后符号位变成 `01` 或 `10`，说明溢出了。
* `01`：正溢出（上溢）。
* `10`：负溢出（下溢）。
判断结果的正负，看**最高位**（第一位符号位），为 1 就是负数，还要进行取反加 1 才能得到真值的绝对值。

**4. 浮点数的加减运算（五步法）**
这个必考！计算步骤：
1. **对阶**：小阶向大阶看齐。尾数随之右移。
2. **尾数相加减**。
3. **规格化**：结果如果不符合 `1.xxx` 的形式，需要左规或右规。
4. **判溢出**：注意，**浮点数是否溢出是由阶码决定的**，尾数溢出只影响精度（可以通过右规解决）。如果阶码全 0 或全 1（比如 8位阶码出现 0 或 255，5位阶码出现 0 或 31），才是真正的浮点数溢出。
5. **舍入**：在对阶或右规时丢弃的位，通过保护位、舍入位、粘滞位（只要后面有非 0 位，粘滞位就置 1）进行判断，决定是“0舍”还是“1入”。

另外，关于硬件电路，不用大家画复杂的并行加法器，但**n位串行可控加减法器**的电路图（实验里做过的，带有异或门控制加减信号的）需要掌握。

### 第四章：指令系统与寻址方式
**1. 寻址方式与有效地址计算（重点：相对寻址）**
相对寻址的有效地址 $EA = PC + 偏移量 D$。
**注意细节**：这里的 PC 指的是**下一条指令的地址**。比如当前转移指令所在的地址是 256，每条指令占 3 个字节，那么取完这条转移指令后，$PC = 256 + 3 = 259$。
如果目标跳转地址是 290，那么偏移量 $D = 290 - 259 = 31$（换算成 16进制是 `1FH`）。
同理，如果是往前跳（比如目标是 110），那就是 $110 - (128+3) = -21$。要把十进制的 -21 转成补码的 16 进制形式（`EB H`），拼接到指令格式中。

对于直接寻址、$EA = D$；对于变址寻址，有效地址 $EA = (变址寄存器 Rx) + D$。这些都要能根据指令的二进制编码拆解并计算出来。

**2. 指令格式设计（扩展操作码）**
必须掌握书上的例题 5.1 及对应的课后作业题。
假设指令定长为 16 位，地址码长度为 6 位：
* **双操作数指令**：需要 2 个地址码（共 12 位），所以操作码占 4 位（$16 - 12 = 4$）。4位操作码最多可以有 $2^4 = 16$ 种编码。
* **扩展机制**：如果双操作数指令只用了 M 条，那么还剩下 $(16 - M)$ 种操作码编码没用。这 $(16 - M)$ 种编码可以向下一个地址段借位扩展，变成单操作数指令。单操作数指令的操作码长度扩展到 10 位（4位 + 6位借来的），所以可以扩展出 $(16 - M) \times 2^6$ 条单操作数指令。
以此类推，一定要会算这种给定 M 条指令，求剩下可以扩展多少条指令的题目。以及计算指令的“平均操作码长度”。

**3. MIPS 汇编指令**
大家需要记住几个核心寄存器的编号：
* `$zero` 是 0号
* `$t0` 是 8号
* `$s0` 是 16号
还需要记住指令的三种格式（R型、I型、J型）的具体划分，以及几个常用指令的 `Opcode` (操作码) 和 `Funct` (功能码)：
* **R型**：`add` (op=0, funct=32 / 20H)，`sub` (op=0, funct=34 / 22H)。
  * 划分格式：`Op(6) | rs(5) | rt(5) | rd(5) | shamt(5) | funct(6)`
  * 注意：汇编代码写的是 `add rd, rs, rt`，但机器码的排列顺序是 `rs, rt, rd`。
* **I型**：`lw` (op=35 / 23H)，`sw` (op=43 / 2BH)，`beq` (op=4)。
  * 划分格式：`Op(6) | rs(5) | rt(5) | immediate(16)`
* **J型**：`j` (op=2)。

**高级应用**：如何用 MIPS 构造数组地址？
比如要访问数组元素 `A[i]`（假设基址在 `$s6`，索引 $i$ 在 `$s3`）：
不能直接用寻址方式获取，必须先算偏移地址：`i * 4`。
1. 用逻辑左移 2 位代替乘 4：`sll $t0, $s3, 2`。
2. 基址加偏移得到元素实际地址：`add $t0, $s6, $t0`。
3. 取出数据：`lw $t1, 0($t0)`。

### 第六章：流水线技术
流水线重点考**数据冒险（数据冲突）**和**控制冒险（控制冲突）**。
作业题一定要搞明白：
1. 分析多条指令之间存在哪些数据相关性（写后读等）。
2. 为了解决冲突，需要插入几个气泡（NOP）或者使用转发（旁路）技术。
3. 计算执行整个程序需要的总时钟周期数。

**时钟周期的计算**：
流水线处理器的时钟周期等于**耗时最长的那一个流水段的延迟**（比如 350ps）。
而在单周期处理器中，时钟周期等于整条指令执行完（5个阶段全加起来）的总耗时。
注意：在流水线中，一条 `lw` 指令虽然每个周期都在往前推进，但它走完 5 个流水段总共还是需要 $5 \times 350ps$ 的时间。

### 第七、十章：存储系统（内存扩展与 Cache）
**1. 存储芯片的判断与引脚计算**
* **SRAM 与 DRAM 的区分**：看引脚。DRAM 没有片选信号（CS），而是用行选通（RAS）和列选通（CAS）代替。SRAM 有片选信号。
* **DRAM 地址线特征**：DRAM 地址线是复用的，所以它外部的地址引脚数量是其内部实际地址位数的一半。
* **容量计算**：容量 = $2^{\text{地址线数量}} \times \text{数据线根数}$。注意单位是位（bit）。

**2. 主存容量的位扩展与字扩展（重点）**
这是必须掌握的大题。给你 CPU 规格和存储芯片规格，要求设计存储器：
* **计算 CPU 地址线**：比如总容量 256KB（$256K \times 8$位），$256K = 2^{18}$，所以 CPU 需要 18 根地址线（A0~A17）。
* **计算所需芯片数量与地址范围**：
  * 末地址 = 首地址 + 容量单元数 - 1。注意是 16 进制的运算。
  * 比如 ROM 区大小是 64KB，单片芯片是 $64K \times 4$位。这就需要 2 片进行**位扩展**（并联数据线）拼成 $64K \times 8$位。地址线需要 16 根（A0~A15）。
  * RAM 区如果是 192KB，单片芯片是 $32K \times 8$位。就需要 $192 / 32 = 6$ 片进行**字扩展**。单片 $32K$ 需要 15 根地址线（A0~A14）。
* **译码器连接（如 3-8 译码器）**：
  CPU 有 18 根地址线，RAM 芯片只用了 15 根。剩下高位的 3 根（A15, A16, A17）就要接在 3-8 译码器的输入端，用来产生各个芯片的片选信号。
  如果 ROM 需要的地址线是 16 根（A0~A15），那么 A15 已经连入 ROM 内部，CPU 剩下的高位只剩 A16 和 A17。这时候如何用译码器同时控制 ROM 和 RAM？可以通过在译码器输出端加逻辑门（比如或门），将多个输出端合并，以匹配 ROM 的地址范围。这个画图连线逻辑一定要理解。

**3. DRAM 的刷新**
作业里做过的集中刷新、分散刷新、异步刷新的计算，要了解死区时间的计算方法。

**4. 高速缓存 Cache**
由于虚存没讲，所以这部分**必考 Cache**。重点掌握：
1. **映射方式**：直接映射、全相联映射、组相联映射。
